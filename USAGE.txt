PodWatcher v1.0.8 — Installation Guide
=======================================

PREREQUISITES
-------------
PodWatcher is a stateless monitoring agent that runs inside your Kubernetes
cluster. It observes pod events and streams notifications to Teams, Slack, or
PagerDuty to reduce MTTR and alert fatigue.

Required tools:
- AWS CLI    : https://docs.aws.amazon.com/cli/latest/userguide/getting-started-quickstart.html
- Helm 3.8+  : https://helm.sh/docs/intro/install/
- kubectl    : Configured with cluster-admin access

AWS Marketplace subscription active.

Notification setup (optional — PodWatcher starts without a webhook):
- Microsoft Teams : https://learn.microsoft.com/en-us/microsoftteams/platform/webhooks-and-connectors/how-to/add-incoming-webhook
- Slack           : https://docs.slack.dev/messaging/sending-messages-using-incoming-webhooks/
- PagerDuty       : Create an Events API v2 integration and copy the routing key


STEP 0 — Confirm you are connected to the correct cluster
----------------------------------------------------------
kubectl config current-context

Verify the output matches your intended cluster. To switch context:
aws eks update-kubeconfig --region <YOUR_REGION> --name <YOUR_CLUSTER_NAME>


STEP 1 — Download the install script
-------------------------------------
curl -O https://raw.githubusercontent.com/saqtek/podwatcher-install/main/helmupgrade.sh


STEP 2 — Authenticate to AWS Marketplace ECR
---------------------------------------------
aws ecr get-login-password --region us-east-1 | helm registry login --username AWS --password-stdin 709825985650.dkr.ecr.us-east-1.amazonaws.com

You should see: Login Succeeded

Note: The ECR token expires after 12 hours. Re-run this command before upgrading.


STEP 3 — Edit the script variables
------------------------------------
Open helmupgrade.sh and set:

  CLUSTER_NAME          — name shown on every alert card (e.g. "prod-us-east-1")
  TEAMS_WEBHOOK         — your Microsoft Teams incoming webhook URL (optional)
  SLACK_WEBHOOK         — your Slack incoming webhook URL (optional)
  PAGERDUTY_ROUTING_KEY — your PagerDuty Events API v2 integration key (optional)
  WATCH_NAMESPACES      — comma-separated namespaces to monitor, or leave blank for all
  SA_NAME               — AWS Marketplace subscribers: leave blank (AWS substitutes automatically)
                          Direct installs (AKS, GKE, on-prem): set to "podwatcher"


STEP 4 — Run the install
-------------------------
bash helmupgrade.sh

This runs: helm upgrade --install with --atomic
If any step fails the install rolls back automatically.


STEP 5 — Verify
----------------
kubectl get pods -n podwatcher
kubectl logs -n podwatcher -l app.kubernetes.io/name=podwatcher --tail=20

Expected: pod status 1/1 Running, logs show "PodWatcher ACTIVE"


STEP 6 — Confirm alerts are working
-------------------------------------
kubectl run test-crash --image=busybox --restart=Always --namespace=default -- sh -c "exit 1"

Wait ~60 seconds. A P2 alert card should arrive in your Teams, Slack, or PagerDuty channel.

Clean up:
kubectl delete pod test-crash --ignore-not-found


DATA SECURITY & RBAC
---------------------
- Data isolation  : Runs entirely within your VPC. Data only leaves via encrypted
                    alert payloads to your configured webhook endpoints.
- Encryption      : All outbound communication uses TLS 1.2+.
- Stateless       : No persistent storage. Events are processed in-memory and
                    purged immediately — nothing written to disk.
- Webhook secrets : Webhook URLs and keys are passed at deploy time and stored
                    as Kubernetes environment variables under your control.
- RBAC read-only  : Dedicated ServiceAccount with least-privilege read-only
                    permissions (get, list, watch) for pods, logs, events,
                    deployments, and nodes — cluster-wide.
- RBAC write      : Write access (create, get, update, patch) scoped exclusively
                    to coordination.k8s.io/leases in the PodWatcher release
                    namespace only. No write access outside that namespace.
                    This satisfies zero-trust and defence-in-depth requirements
                    for enterprise and regulated environments.
- Container       : Non-root (UID 10001), read-only root filesystem, all Linux
                    capabilities dropped, seccomp RuntimeDefault.


RESOURCE QUOTAS
---------------
- Memory limit  : 512Mi
- CPU limit     : 400m (0.4 vCPU)
- Memory request: 128Mi
- CPU request   : 100m

Peak observed under 100 simultaneous pod crashes: 87Mi memory / 45m CPU.


UPGRADING
---------
1. Re-run STEP 2 (ECR login — token expires after 12 hours)
2. Update the --version flag in helmupgrade.sh to the new version
3. Re-run: bash helmupgrade.sh


UNINSTALL
---------
helm uninstall podwatcher -n podwatcher
kubectl delete namespace podwatcher
